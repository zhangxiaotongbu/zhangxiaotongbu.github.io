一、git介绍

Git：分布式版本控制工具

分布式版本控制工具VS集中式版本控制工具

集中式版本控制工具：SVN…

在一台服务器上进行存放项目历史版本

缺点：如果服务器出现宕机，则无法历史版本的记录

分布式版本控制工具：Git…

克服了集中式版本控制工具的缺点

每台电脑都会存放项目的历史版本

Git结构

代码托管中心（远程库）

局域网环境下：GitLab（公司内部自己搭建服务器，不对外公开）

互联网环境下：GitHub和Gitee（码云）

二、Git命令介绍

命令名称 作用
git config –global user.name 用户名 设置用户签名
git config –global user.email 邮箱 设置用户签名
git init 初始化本地库
git status 查看本地库状态
git add 文件名 添加到暂存区
git commit -m “日志信息” 文件名 提交到本地库
git reflog 查看精简历史记录
git log 查看详细历史记录
git reset –hard 版本号 版本穿梭
